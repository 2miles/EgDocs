SD-cards

Copying an SD-card

I did this on a Mac using ApplePi Baker. All the infor for the application can be found here: ApplePi-Baker

This is the tutorial I followed El tutorial

Using ApplePi Baker:

Copying a SD to an External HD as a .img

  1. Choose the SD card form the leftmost box
  2. Choose BACKUP, the middle box (Image to file)
  3. Choose the HD drive to copy to
  4. Name the file backupBlaBla.img
  5. Choose ISO filetype
  6. Click Start

Copying a .img in a HD to a SD card.

  1. Choose the SD card from the leftmost box
  2. Choose the RESTORE, the rightmost box (file -> image)
  3. Choose the filename form the HD
  4. Click Start.

Adding & Removing Games

Adding games with USB stick

This is the archaic method. It is faster than over wifi though.

  1. Make sure you have a USB stick ready to transfer games.
    1. Stick a USB stick into Just drag and drop the ROMS into the USB
    2. Format any empty USB stick to FAT32, and make an empty folder called 'retropie'
    3. Plug that USB into the Pi. Wait a minute, and it will populate the retropie folder with a few sub-folders. Plug it back into the computer.
    4. Now go into the newly created roms folder in the retropie folder. Inside it will be a list of folders, one for each console. This is where we can drag and drop games.
  2. Insert USB into computer and drag the ROMS into the corrisponding console folder in retropie/roms/console. Wait a little for them to transfer. Now the games are on the stick.
  3. Insert USB stick into the Pi, top left slot is best I think. Wait a few mins for the games to transfer and now you can unplug the stick from the Pi. (note:) the USB only adds games to the Pi. So you are free to delete the games on the USB and plug it back into the Pi and it wont remove games from the Pi.

Adding/removing using SSH over wifi

This method is the slowest but by far the most convenient and easiest on the hardware. As long as the Pi is on and you're on the same wifi then you can transfer remotely wich is really convenient. You can even play it while it is transfering.

  1. If you havent already done it yet, SSH must be enabled on the Pi. You can see how to do that here: Enable SSH on Pi
  2. Get a program that can use SSH and open it up. I like FileZilla.
  3. Enter the connection info in the fields in the top bar. You'll need the address, user name, and password of the Pi, and the port number, which is 22. Go here on how to get that info SSH and wifi
  4. The left side will be your computer file tree, the right side, when connected, will be the Pi's file tree.
  5. To add, you just drag and drop the files from your computer to the correct location on the Pi:
    /opt/retropie/roms/consoleName/
    To remove, you just click on the game in the Pi and delete it.
  6. When the upload is complete, restart the Pi and you're good to go.

Adding/removing using the SD-card

This method is the fastest and most direct method. We take the SD out of the Pi and insert it directly into the computer. Since computers dont usualy take micro-SD cards this requires either a USB to microSD converter or a microSD to SD converter, depending on what ports your computer has.

  1. Remove micro-SD card from Pi and insert it into the computer.
  2. To add, just drag the games from the computer directly into
    /opt/retropie/roms/consoleName/
    To remove, just click on the games to delete them.
  3. Eject the SD card and put it back into the Pi. Now you are good.

Adding consoles to EmulationStation

If the emulator is not already in the roms list, then create the required rom folder name. The name must match exactly. You can find the emulator names here, just Click on the dropdown menu and got to emulators, here you visit the page of you rsystem to find the shorthand name.

  1. Create required short-hand rom folder, put it with other rom folders.
  2. Downliad the required emulator:
    RetroPie → rpsetup → manage packages → manage optional packages → choose emulator from the list.
  3. Choose download form source
  4. put roms into the new rom folder, if you want any games..
  5. Runscraper on the new rom folder for console meta-data:
    scraping

Scraping & Metadata

When the scraper looks fro metadata(images, etc..) it checks for the gamelist.xml file in

/opt/retropie/config/all/emulationstation/gamelist/consoleName/

If it is not there then it will create one. The scraper places the images it finds into:
/opt/retropie/config/all/emulationstation/downloaded_images/consoleName/

  • Make sure there are no other gamelist.xml files lay8ing aroud, that will mess things up.
  • If the scraper is no working just wait a few hours/days. It goes down form time to time. Plus, I think there may be a daily scrape limit.

Scraping with EmulationStation

Access the scraper from the EmulationStation menu.

Start → scraper → use screenscraper → scrape now → select systems, choose missing images, → back → user decides conficts: no → then scrape

Definitions

addon (Kodi)
Program or app installed within Kodi. They host the content that users stream on their device.
ApplePi Baker
This application is for MacOS X only and allows you to prepare an SD-Card for use with Raspberry Pi’s NOOBS, and it allows you to “flash” and IMG file to an SD-Card.
ArcOS
a Linux distribution based on Arch Linux, designed primarily for hosting web sites and services on cheap ARM-based devices such as Raspberry Pi
BIOS
"Basic Input/Output System" In most classic system is the firmware that bootstraps the user software (cartdridges).
EmulationStation
The official front-end of RetroPi project; a game launcher with controler/key support, themes, scrapers, game metadata.
Etcher.io
Etcher can quickly flash an operating system into an SD card. Etcher is used for writing image files zipped folders creating USB flash drives that are bootable.
flashing
To put an OS onto an SD card.
gamelists
xml file with all the game meta data and links to each games artwork.
kodi
Free and open-source media player software application
Libretro
An API.. A program is converted to a library file ‘libretro core’. A frontend (RetroArch) that supports the API can load an run the core. The frontend’s responsibility is to provide implem. specif. details like video/audio/input drivers. The libretro core’s responsibility is solely to provide the main program.
meta-key
To access RetroArch from Emulationstation. Manage saves, controlers, reatart system.
overclock
run processor at a higher speed than intended by manifacture.
Raspberry Pi
A series of small single-board computer from the UK.
Raspberry OS
Formally Raspbian, a Debian-based OS for the Pi.
Real Debrid
Is a subscription service that provides access to additional file hosters.
RetroArch
A front-end for emulators (libretro), game engines, and media players.
RetroPie
A free linux distro OS for launching retro games, by packaging togeter things like EmulationStation and
RetroArch
RetroArch is a frontend for emulators, game engines and media players
ROMs
